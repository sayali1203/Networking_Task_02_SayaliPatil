Part A-Network Devices Research

Router
Purpose: Connects different networks and provides internet access.
How it Works: Receives data packets and forwards them to the correct destination using IP addresses.
Real-World Usage: Home Wi-Fi routers.

Switch
Purpose: Connects devices inside the same network.
How it Works: Uses MAC addresses to send data to the correct device.
Real-World Usage: Offices connecting multiple computers.
Hub
Purpose: Connects devices in a LAN.
How it Works: Broadcasts data to all connected devices.
Real-World Usage: Older small networks.

Access Point
Purpose: Provides wireless connectivity.
How it Works: Connects wireless devices to a wired network.
Real-World Usage: Wi-Fi in schools and offices.

Firewall
Purpose: Protects the network from unauthorized access.
How it Works: Filters incoming and outgoing traffic.
Real-World Usage: Windows Firewall.

Modem
Purpose: Connects home network to ISP.
How it Works: Converts digital signals to transmission signals.
Real-World Usage: JioFiber, Airtel Xstream modem.

What role does your router play?

What would happen if DNS stopped working?
Websites would not open using names like google.com because domain names could not be translated into IP addresses.

Part D- Network Communication Flow

Your Device
When a user enters www.google.com⁠, the device sends a request to find the website.

Router
The router forwards the request to the internet.

DNS Server
DNS translates google.com into an IP address.

Google Server
The request reaches Google's server and the webpage data is prepared.

Response
The webpage data returns through the network and appears in the browser.
